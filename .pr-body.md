## PR: fix: add UQPay recharge infrastructure tables and webhook skeleton

### 变更内容

| # | 变更 | 说明 |
|---|------|------|
| 1 | `cards.balance_id` 字段 | UQPay 平台余额 ID，用于卡充值扣减平台 UQPay 余额。自动迁移，已存在则跳过 |
| 2 | `uqpay_recharge_orders` 表 | 充值订单表，状态机：`PENDING / SUCCESS / FAILED / REFUNDED / CANCELLED` |
| 3 | `uqpay_webhook_events` 表 | Webhook 事件记录，幂等 `event_id` 唯一约束 |
| 4 | `POST /api/v1/webhook/uqpay/notify` | UQPay Webhook 事件接收路由，幂等写入，敏感字段过滤 |
| 5 | `GET /api/v1/webhook/uqpay/events` | 管理端查询 Webhook 事件记录（分页） |
| 6 | SDK `listAccounts()` | 已在 `uqpay.ts` 存在，PR-2 可直接使用获取 `balance_id` |

### 安全性

- 不保存 PAN/CVV 等敏感卡号数据
- 不输出 token / clientId / apiKey
- 不输出完整 webhook payload 中的敏感字段
- Webhook payload 深度过滤（已知敏感字段替换为 `[REDACTED]`）
- `event_id` 幂等：重复事件直接返回 SUCCESS，不重复写入

### 本轮约束

- 不调用 UQPay `POST /api/v1/cards/recharge` 真实充值接口
- 不真实充值
- 不修改钱包余额
- 不修改卡余额
- 不做资金入账/出账处理（PR-3 后续处理）

### 构建验证
